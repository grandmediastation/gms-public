# Contributing to Grand Media Station

Thanks for helping make GMS better. The application source is private, so
contributions here are **bug reports, feature requests, and feedback** rather
than code pull requests.

## Before you open an issue

1. **Search first.** Check [open and closed issues](../../issues?q=is%3Aissue)
   for an existing report. If you find one, add details or a 👍 instead of
   opening a duplicate.
2. **Use a template.** Pick *Bug report* or *Feature request* — the fields help
   triage faster.

## Privacy & security — please read

GMS is built privacy-first: your library and personal data never leave your
device. Keep your reports the same way.

- **Scrub before you paste.** Remove file paths, license keys, email addresses,
  tokens, and IP addresses from logs and screenshots.
- **Never post a security vulnerability as a public issue.** Use the private
  advisory link on the *New issue* screen (or Security → Advisories). We'll
  coordinate disclosure with you.

## What to expect

GMS is built and maintained by a solo developer, so responses may take a few
days. Clear, reproducible reports get resolved fastest. Feature requests are
weighed against the roadmap; a request being declined isn't a judgment on the
idea.

## Issue etiquette

- One problem per issue.
- A reproducible bug beats a vague description — exact versions, OS, and steps.
- Be civil. This is a small project run by a real person.
